AI Assistant API Documentation

Version 2.1.0

Base URL

https://yourproject.supabase.co/functions/v1

Authentication

All endpoints require authentication via Supabase JWT or Bearer token.

Authorization: Bearer [your_jwt_token]

Response Format

All responses are in JSON format:

{
  "data": { ... },
  "status": "success"
}

Error Handling

Standard error format:

{
  "error": true,
  "code": 401,
  "message": "Unauthorized"
}

Rate Limits

Free Tier

60 requests per minute per endpoint

Pro Tier

200 requests per minute

When rate limited, check the Retry-After header for wait time.

Realtime Updates

Subscribe to realtime changes using Supabase Realtime:

tasks

Create/update/delete events

chat_messages

New messages in active sessions

calendar_events

Event changes and reminders

Authentication Endpoints

These endpoints are handled by Supabase Auth with additional validation.

POST /auth/signup

Register a new user account

POST /auth/login

Authenticate and receive JWT

GET /auth/user

Fetch current user profile

Example Response

{

  "email": "user@example.com",
  "preferences": {
    "theme": "dark",
    "assistant_style": "professional"
  }
}

Task Management

Endpoints for creating, reading, updating, and deleting tasks.

GET /tasks

Fetch all tasks for the authenticated user

Query Parameters

status

Filter by status (todo, in_progress, done)

priority

Filter by priority (low, medium, high)

due_before

ISO date string for filtering

updated_since

Get tasks updated after timestamp

POST /tasks

Create a new task

Request Body

{
  "title": "Buy milk",
  "description": "Almond only",
  "priority": "medium",
  "status": "todo",
  "due_date": "2024-04-10T10:00:00Z",
  "source_type": "manual"
}
PUT /tasks/:id

Update an existing task

Notes

Partial updates are supported. Only include fields you want to change.

DELETE /tasks/:id

Delete a task

Calendar Management

Endpoints for managing calendar events and scheduling.

GET /calendar

Get upcoming events within a date range

Query Parameters

range_start

Start of date range (ISO format)

range_end

End of date range (ISO format)

POST /calendar

Add new event manually or via LLM

Request Body

{
  "title": "Doctor Appt",
  "start_time": "2024-04-12T14:00:00Z",
  "end_time": "2024-04-12T15:00:00Z",
  "location": "Clinic",
  "source": "llm"
}
DELETE /calendar/:id

Delete an event

Chat Assistant

Endpoints for interacting with the AI assistant.

POST /chat

Send user prompt to assistant

Request Body

{
  "session_id": "abc-123",
  "message": "What do I have this week?",
  "model": "phi3-mini",
  "mode": "calendar"
}

Response

{
  "reply": "You have 3 events this week...",
  "actions": [
    {
      "type": "show_calendar",
      "data": { ... }
    }
  ]
}

Sync Utilities

Endpoints for synchronizing data between devices.

GET /sync/tasks

Fetch only updated tasks for mobile sync

Query Parameters

updated_since

ISO timestamp of last sync

POST /sync/bulk

Upload batch of offline-edited tasks/messages

Request Body

{
  "tasks": [
    {
      "id": "task-123",
      "title": "Updated task",
      "updated_at": "2024-04-10T12:00:00Z"
    }
  ],
  "messages": [
    {
      "session_id": "sess-456",
      "content": "Hello from offline",
      "created_at": "2024-04-10T12:05:00Z"
    }
  ],
  "conflict_strategy": "last-write-wins"
}

Conflict Strategies

last-write-wins

Most recent update is kept

prompt-user

User resolves conflicts manually

Backend Overview

The backend supports a privacy-conscious AI assistant for mobile platforms. It handles user authentication, task and event management, voice/text interaction routing, and LLM integration.

Designed with mobile-first principles, it ensures fast response times, hybrid reasoning, and local fallback capabilities.

Mobile-First

Optimized for low-latency mobile interactions with efficient data transfer

Privacy Focused

Local processing options and minimal data retention policies

Hybrid Architecture

Seamless switching between cloud and local models based on context

Technology Stack

FastAPI Python Framework
PostgreSQL Database
JWT Authentication
RESTful API Style
MLC LLM Model Serving
Whisper.cpp Speech Processing
Pydantic Validation
llama.cpp On-device LLM

Core Features

User Management

POST /auth/register

Create a new user account with email and password

Request Body: 
{
  "email": "string",
  "password": "string",
  "name": "string"
}
POST /auth/login

Authenticate and receive JWT token

Request Body: 
{
  "email": "string",
  "password": "string"
}
GET /auth/me

Fetch current authenticated user profile

Headers: 
Authorization: Bearer {token}

Tasks & Calendar

GET /tasks

List all tasks for authenticated user

Query Params: 
{
  "status": "pending|completed",
  "due_date": "YYYY-MM-DD"
}
POST /tasks

Create new task (supports NLP input)

Request Body: 
{
  "title": "string",
  "description": "string",
  "due_date": "YYYY-MM-DD",
  "nlp_input": "string" // e.g., "Buy milk tomorrow"
}
POST /calendar/parse

Parse natural language into structured event

Request Body: 
{
  "text": "Meeting with team at 3pm next Tuesday"
}

Conversational Assistant

POST /chat

Send message to LLM and get response

Request Body: 
{
  "message": "string",
  "context": "string", // optional
  "model_preference": "cloud|local" // optional
}
POST /reason

Hybrid routing for complex reasoning tasks

Request Body: 
{
  "task": "string",
  "context": "object" // optional additional context
}

Voice Interaction

POST /voice/transcribe

Transcribe audio using Whisper.cpp

Request Body: 
{
  "audio": "base64_encoded_audio",
  "language": "en" // optional
}
POST /voice/command

Convert transcription to executable command

Request Body: 
{
  "transcription": "string",
  "context": "object" // optional app context
}

Background Jobs

Daily Summarization

Cron job that runs at midnight to summarize the day's events and tasks

Notification Triggers

Checks every 15 minutes for upcoming events/tasks to trigger notifications

Model Cache Updates

Weekly updates to local model cache when connected to WiFi

Error Handling

Standard Error Schema

{
  "error": {
    "code": "string", // e.g., "auth/invalid-credentials"
    "message": "string", // Human-readable message
    "details": "object|null" // Additional error context
  }
}

Auth Errors (4xx)

401 Unauthorized 403 Forbidden 429 Too Many Requests

Model Routing Errors (5xx)

503 Service Unavailable 504 Gateway Timeout

NLP Parser Fallback

When NLP parsing fails, the system falls back to structured input with clear error messages about which fields are required.

Security

JWT Authentication

All routes except login/register require valid JWT in Authorization header

Pydantic Validation

All input data strictly validated with Pydantic models

Rate Limiting

LLM endpoints have strict rate limits to prevent abuse

Encryption Module

Optional homomorphic encryption for sensitive PII fields

Privacy Levels

Local Only - All processing on device
Hybrid - Non-sensitive data to cloud
Cloud Preferred - Maximum capabilities
